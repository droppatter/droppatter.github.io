静态博客，但是复健。

疫情居家，得闲难耐。想起六年前给自己和安琪做过博客，参考了UP主搬运崽啊的视频，尝试在Win和Mac系统上重新搭建。

静态博客工具

  • 安装Node.js 一个JavaScript的运行环境。

参考文章: Node.js安装及环境配置之Windows篇-刘奇云

  • 安装Git 一个管理系统。

参考文章：Windows系统Git安装教程（详解Git安装过程）-学为所用

在管理员运行的CMD中输入：

node -v            # 这三个都是检查版本，来查看是否安装成功
npm -v
git --version
  • 安装Hexo

在管理员运行的CMD中输入：

npm install hexo-cli -g     # 安装Hexo
hexo -v                     # 检查版本

GitHub建仓生成SSH Keys

  • 生成SSH Keys

打开任意文件，右键打开Git Bash，输入：

> git config --global user.name "GitHub账号的名字"，
> git config --global user.email "GitHub账号的邮箱"
> ssh-keygen -t rsa -C "GitHub账号的邮箱"                  # 按3次回车即可。
> cat ~/.ssh/id_rsa.pub                                 # 输出SSH Key全部复制下来~
  • 导入SSH Keys

在C盘-用户-用户名-.ssh中找到id_rsa.pub,用编辑器打开，全选复制内容；在GitHub的Setting-SSH and GPG keys-SSH keys中新建导入。

  • 测试是否绑定

在Git Bash中输入：

ssh -T git@github.com

预览生成发布博客

  • GitBash初始化Hexo

总是报错，尝试了下GitBash加速npm，本质是换源到淘宝的npm镜像网站。参考：https://www.likecs.com/show-204877074.html

发现原来是管理员权限的问题，需要cd到当前文件夹。后续补充一下管理员权限打开GitBash Here。但是换源肯定不亏嗯嗯..

>hexo init                            #初始化Hexo
>hexo s                               #生成本地博客，可以当预览
>npm install hexo-deployer-git --save #安装发布工具，一次即可
>hexo g    							  #生成博客
>hexo d   							  #上传发布`
